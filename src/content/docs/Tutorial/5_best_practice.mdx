In diesem Kapitel werden praktische Beispiele für Barrierefreiheitstests von Webseiten und Webanwendungen vorgestellt. Es werden typische Probleme aufgezeigt, die während solcher Tests auftreten können, sowie Lösungsansätze für eine verbesserte Zugänglichkeit.

Testszenarien:
1. Formulare: Identifizieren und beheben Sie Probleme wie unklare Formularelemente, fehlende Beschriftungen oder Schwierigkeiten bei der Tastaturnavigation.
2. Bilder: Überprüfen Sie alternative Texte für Bilder und stellen Sie sicher, dass sie angemessen beschrieben sind, um eine korrekte Interpretation durch Screenreader zu ermöglichen.
3. Farbkontrast: Testen Sie den Farbkontrast von Texten und Hintergründen, um sicherzustellen, dass sie für Menschen mit Sehschwierigkeiten ausreichend lesbar sind.
4. Multimedia-Inhalte: Prüfen Sie die Zugänglichkeit von Videos und Audiodateien durch die Bereitstellung von Untertiteln, Transkripten oder alternativen Darstellungsformen wie zum Beispiel Gebärdenspracvideo (DGS - deutsche Gebärdensprache).

- Problem: Ein Bild enthält keinen alternativen Text.
 - Lösung: Fügen Sie einen beschreibenden alternativen Text hinzu, der den Inhalt des Bildes präzise wiedergibt.
- Problem: Ein Formularfeld hat keine Beschriftung.
 -- Lösung: Fügen Sie eine klare Beschriftung hinzu, um den Zweck des Formularfelds zu erläutern und eine intuitive Navigation zu ermöglichen.
- Problem: Der Farbkontrast zwischen Text und Hintergrund ist zu gering.
 -- Lösung: Passen Sie die Farben so an, dass ein ausreichender Kontrast gewährleistet ist, um die Lesbarkeit für alle Benutzer zu verbessern.
- Problem: Ein Video hat keine Untertitel.
 -- Lösung: Fügen Sie Untertitel hinzu, damit hörbeeinträchtigte Benutzer den Inhalt des Videos verstehen können.

Die Beispiele vermitteln ein tieferes Verständnis für Barrierefreiheit, sodass alle, die an der Gestaltung von Webseiten beteiligt sind, lernen können, wie sie ihre Inhalte für eine breitere Nutzergruppe zugänglich machen.
