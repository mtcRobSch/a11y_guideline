# Teststrategie - Entwurf
- manuelle Testung kann anhand von Kriterien nach einer Checkliste erfolgen

## Erste Sichtung (visuell)

### Wahrnehmbarkeit

* Kontrast zwischen Vordergrund (Text, Bilder, Buttons, ...) und Hintergrund mindestens 4,5:1
* Links sichtbar (unterstrichen, andere Farbe)
* kein automatischen Slider/ Pop-up (Option deaktivieren)
* Erkennbarkeit und Benutzbarkeit von Buttons (Sehen und Verstehen)
* Animationen (Option deaktivieren)
* hat die Seite Einstellungen für Barrierefreiheit (optionale Anpassungen)

o leichte Sprache

o DGS

o Schriftgrößen verändern

o Vorlesefunktion



## Layout

* Übersichtlichkeit
  keine automatischen Slider/Pop-up (Option zum deaktivieren)
  Schritgröße
  Zeilenabstand
  gut lesbare Schriftart

### Kontrast

* zwischen Text und Hintergrund
  zwischen Text und Bild

### Links erkennbar

* unterstrichen
  andere Farbe
  hervorheben

### Buttons eindeutig/Erkennbar

* Größe
  Farbe
  eindeutige Formulierung

## Bedienung

### Maus

* Inhalte mit der Maus fokussieren
  Benötige ich Mausgesten? (z.B. drag\&drop, Menubedienung)
  Sind Click-Flächen groß genug? (min. In gleicher Textgröße)

### 1.2.2 Tastatur

- Tab-Taste

o Hervorhebung aktueller Fokus

o Kann ich durch Seite springen über alle wichtigen Seitenbestandteile (Überschriften, Links, Buttons, ..)

- Leertaste

o Kann ich Menus/Aufklapplisten aufklappen

o Kann ich bei Formularen/Checkboxen etc. An-/abwählen

o Kann ich Toggle-Schalter wechseln

- Enter-Taste

o Kann ich Links besuchen, wenn vorher Fokus drauf liegt durch Tab

o Kann ich Buttons auswählen/bestätigen, wenn Fokus vorher darauf liegt mit Tab

### Screenreader

<u>Kapitel wird sehr ausführlich, wenn Bedienung von Screenreader aufgeführt</u>

- Tastatur wie unter 1.2.2

- Cursor up/down

o Kann ich durch den Seiteninhalt lesen von Absatz zu absatz

- Weitere Kurztasten für Überschriften, Listen, Links, Buttons, Tabellen, Abschnitte/Sections
